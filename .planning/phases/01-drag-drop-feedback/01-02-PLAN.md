# 01-02: Drop Zone Pulse Animation

## Goal
Add visual feedback to columns when a draggable task hovers over them (drop zone indicator).

## Context
Columns need to show they're valid drop targets. Currently, there's no visual indication when dragging over a column.

## Implementation

### Step 1: Add pulse animation to globals.css

**File: `src/app/globals.css`**

Add to `@layer utilities` section (after line 118):
```css
/* Drop zone pulse animation */
@keyframes drop-zone-pulse {
  0%, 100% {
    border-color: rgba(168, 85, 247, 0.3);
    box-shadow: 0 0 0 0 rgba(168, 85, 247, 0);
  }
  50% {
    border-color: rgba(168, 85, 247, 0.6);
    box-shadow: 0 0 20px 0 rgba(168, 85, 247, 0.3);
  }
}

.drop-zone-active {
  animation: drop-zone-pulse 1.5s ease-in-out infinite;
}
```

## Verification

- [ ] When dragging a task over a column, border pulses purple
- [ ] Background gets subtle purple tint when active
- [ ] Animation stops when task leaves the drop zone
- [ ] Existing hover effect (border-white/20) still works when not dragging
- [ ] TypeScript compiles without errors

## Notes

- The pulse animation uses 1.5s duration for a gentle, non-distracting effect
- `isOver` state is automatically managed by @dnd-kit
- The `drop-zone-active` class is conditionally applied only during drag-over
