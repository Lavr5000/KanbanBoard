# 01-03: Spring Drop Animation

## Goal
Add spring physics to the drop animation for a more natural, bouncy feel.

## Context
Current drop animation uses default easing. Spring physics feel more premium and match the glassmorphism aesthetic.

## Implementation

### File: `src/widgets/board/ui/Board.tsx`

**Location:** Around line 97-103 (after sensors definition)

**Add useMemo for drop animation config:**
```tsx
// Spring physics drop animation
const dropAnimationConfig = useMemo(() => ({
  duration: 300,
  easing: 'cubic-bezier(0.34, 1.56, 0.64, 1)', // Bouncy spring effect
  sideEffects: defaultDropAnimationSideEffects({
    styles: {
      active: {
        opacity: "0.5",
        transform: 'scale(1.05)',
      },
    },
  }),
}), []);
```

**Update DragOverlay to use the config (around line 490):**
```tsx
<DragOverlay dropAnimation={dropAnimationConfig}>
  {activeTask && <DragPreviewTaskCard task={activeTask} isDragging={true} />}
</DragOverlay>
```

**Note:** `useMemo` is already imported from React (line 3).

## Verification

- [ ] Drop animation has a slight bounce
- [ ] Active dragged card shows at 105% scale
- [ ] Animation feels springy, not linear
- [ ] TypeScript compiles without errors
- [ ] No visual glitches on drop

## Notes

- The easing curve `cubic-bezier(0.34, 1.56, 0.64, 1)` creates a bounce effect
- The second value > 1 creates the "overshoot" characteristic of springs
- Duration of 300ms is snappy but not too fast
