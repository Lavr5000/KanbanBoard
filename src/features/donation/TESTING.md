# Тестирование функции донатов

## Чек-лист для ручного тестирования

### 1. Проверка отображения кнопки доната
- [ ] Кнопка "Поддержать проект" отображается в левой панели
- [ ] Кнопка находится над кнопкой "Экспортировать данные"
- [ ] Кнопка имеет розово-фиолетовый градиент
- [ ] Иконка сердца отображается корректно
- [ ] При наведении кнопка увеличивает тень и меняет цвет

### 2. Проверка модального окна
- [ ] При клике на кнопку открывается модальное окно
- [ ] Заголовок "Поддержать проект" отображается
- [ ] Описательный блок с иконкой сердца виден
- [ ] Кнопка закрытия (крестик) работает
- [ ] Клик вне модального окна НЕ закрывает его (по дефолту в Modal)

### 3. Проверка выбора суммы
- [ ] 6 предустановленных сумм отображаются (100, 300, 500, 1000, 2000, 5000)
- [ ] Клик по сумме выделяет её градиентом
- [ ] Можно ввести кастомную сумму в поле ввода
- [ ] При вводе кастомной суммы выделение с предустановленных сумм снимается
- [ ] Выбранная сумма отображается внизу модального окна

### 4. Проверка способов оплаты
- [ ] Три способа оплаты отображаются: СБП, ЮMoney, Тинькофф
- [ ] Каждый способ имеет иконку (эмодзи)
- [ ] По умолчанию выбран СБП
- [ ] При клике на способ оплаты он выделяется градиентом и рамкой
- [ ] Переключение между способами работает плавно

### 5. Проверка QR-кода
- [ ] QR-код отображается для каждого способа оплаты
- [ ] QR-код имеет белый фон и округлённые углы
- [ ] При смене способа оплаты QR-код обновляется
- [ ] Если QR-код не загружается, показывается fallback текст

### 6. Проверка информации для копирования
- [ ] Для СБП отображается номер телефона
- [ ] Для ЮMoney отображается номер счёта
- [ ] Для Тинькофф отображается номер карты
- [ ] Кнопка копирования работает для каждого поля
- [ ] При успешном копировании иконка меняется на галочку
- [ ] Галочка исчезает через 2 секунды

### 7. Проверка адаптивности
- [ ] Модальное окно корректно отображается на десктопе
- [ ] Модальное окно корректно отображается на планшете
- [ ] Модальное окно корректно отображается на мобильном
- [ ] QR-код масштабируется корректно на всех устройствах

### 8. Проверка UX
- [ ] Все тексты на русском языке и читаемы
- [ ] Цвета соответствуют общей теме приложения (тёмная тема)
- [ ] Анимации плавные и не раздражают
- [ ] Кнопка доната выделяется и привлекает внимание
- [ ] Интерфейс интуитивно понятен

### 9. Проверка технических аспектов
- [ ] Нет ошибок в консоли браузера
- [ ] Состояние модального окна управляется через Zustand
- [ ] При закрытии модального окна состояние сбрасывается корректно
- [ ] Компоненты не вызывают ре-рендеров без необходимости

### 10. Проверка интеграции
- [ ] Кнопка доната не конфликтует с другими кнопками в сайдбаре
- [ ] Модальное окно отображается поверх всех других элементов (z-index)
- [ ] Стили не конфликтуют с существующими компонентами
- [ ] FeedbackModal и DonationModal могут открываться независимо

## Сценарии использования

### Сценарий 1: Быстрый донат через СБП
1. Пользователь видит яркую кнопку "Поддержать проект"
2. Кликает на неё
3. В модальном окне по умолчанию выбран СБП
4. Пользователь выбирает сумму 500₽
5. Сканирует QR-код в приложении банка
6. Закрывает модальное окно

### Сценарий 2: Донат через ЮMoney с кастомной суммой
1. Открывает модальное окно
2. Переключается на ЮMoney
3. Вводит кастомную сумму 750₽
4. Копирует номер счёта через кнопку копирования
5. Переходит на сайт ЮMoney и вставляет номер
6. Закрывает модальное окно

### Сценарий 3: Просмотр вариантов без доната
1. Открывает модальное окно из любопытства
2. Просматривает все способы оплаты
3. Читает описание проекта
4. Закрывает окно без действий

## Известные ограничения
- QR-код генерируется через внешний API (qrserver.com)
- Требуется интернет-соединение для загрузки QR-кода
- Данные для оплаты (номера телефонов, карт) в коде захардкожены и должны быть обновлены на реальные
