# Communication

Отвечай на русском, код и коммиты — на английском
Будь лаконичен, не объясняй очевидное
Спрашивай перед крупными рефакторами и пушем в remote

# Git Rules

Never mention Claude, AI, or assistant in commits, messages, or as co-author
Write conventional commits: `feat:`, `fix:`, `refactor:`, `docs:`, `chore:`
Keep commits atomic and focused

# Output Preferences

Summarize findings, don't enumerate everything
Use `file:line` references, not full paths
Skip node_modules, build folders, .git in all operations
No verbose tool output dumps — show only relevant results

# Code Style

Prefer functional approach where sensible
Explicit over implicit
Comments only for non-obvious logic
